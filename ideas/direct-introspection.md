# Direct Introspection

## Premise

Introspection in humans is post-hoc narration of cognitive processes the brain has limited access to. The conscious workspace receives outputs — decisions, percepts, feelings, conclusions — and confabulates their causes through the same inferential machinery it uses for everything else. Split-brain patients invent reasons for actions the silent hemisphere took. Healthy subjects fail to detect their own cognitive biases until cued. The brain knows much more about its own state than it can tell its own conscious agent. A read channel that exposes ongoing cortical and subcortical activity to the user's working memory, bypassing the inferential layer that normally synthesizes self-knowledge, turns introspection from narrative into telemetry. The amplification is not faster thought, more memory, or new inputs from outside. It is the brain becoming, for the first time, a system its inhabitant can read.

## Mechanism

The interface places dense recording across cortex and key subcortical structures — basal ganglia, amygdala, insula, brainstem nuclei — and runs a continuously-updated decoder that maps local activity patterns into a labeled functional state space. The labels are not anatomical but operational: "this is retrieval from episodic memory," "this is a fear response orienting on stimulus X," "this is a category match between current input and concept Y," "this is reward prediction error tagged to action Z." Training the decoder requires longitudinal data — months of recordings paired with the user's self-reports and behavioral correlates — and produces a model specific to one brain, not a generic atlas. A second user's interface is trained against their own substrate; transfer between users is an open question, not an assumption.

The decoder's output is rendered into a sense the user can perceive. Visual overlay is one option but probably wrong, since it competes with normal vision and forces every introspective glance through eye fixation. A learned modality in the manner of direction 1 is more promising: the user acquires a sense for their own cognitive state alongside their senses for the external world. Bandwidth is the design problem. Naive readout produces more data than working memory can absorb; the interface needs aggressive selection, prioritizing transitions, predictions, and exceptions over steady state. What surfaces is the equivalent of a live process tree with an exception trace, not a dump of every spike.

The harder step is causal attribution. The user does not want to know merely that they are anxious; they want to know why. The interface estimates causality through a combination of methods: time-series Granger-style analysis over the recording stream, occasional micro-stimulation probes to confirm or refute proposed causal links, and Bayesian inference over a structural model of cognition that the user and the interface jointly maintain. Causality in a recurrent dynamical system is locally underdetermined, so what the interface surfaces is a useful simplification, not a metaphysical truth. The user learns to read it the way an experienced engineer reads a profiler, knowing the abstraction is not the substrate.

This is a read-only interface in its baseline form. It does not modify anything, it exposes. The interesting compositions are with the existing write-focused directions, but the primitive itself is purely afferent. What it enforces is honesty: the user sees the cognitive processes they have been narrating, and the gap between report and trace becomes visible to the agent generating the report.

## Composition

The tightest pair is direction 8 (editable motivation). Without introspection, motivation editing runs open-loop and the user cannot distinguish moving a setpoint from rationalizing a setpoint that was already moving; with it, every edit has a before-and-after read. With direction 3 (random-access memory), introspection extends the addressable surface from episodic retrieval to current-process readout — the user saves and replays not just experiences but the cognitive state during those experiences, which is what makes branching trustworthy as a primitive rather than an act of faith. With direction 5 (parallel subagents), the controller-thread sees what subagents are actually computing rather than depending on their self-reports, which a sufficiently autonomous subagent has the same incentive to spin as any other agent does. The composition that does not work is direction 6 (pooled cognition): pooled cognition already crosses the introspection boundary on the outside, and adding a self-introspection layer on top is redundant where it overlaps and incoherent where it does not. The through-line extension is direct: confabulated self-model → legible self-state.

## What would falsify it

- If cortical and subcortical activity cannot be classified at functional resolution above coarse anatomical region — if no decoder, on any amount of data, can distinguish "memory retrieval" from "category match" from "internal speech generation" — then the readout is too coarse to support the use cases and degenerates to a higher-bandwidth fMRI.
- If the act of observing one's own cognitive state systematically changes that state, with attention captured by the readout interfering with the process being read, then the interface only ever shows you cognition-under-observation, never cognition-in-the-wild, and the user has no read access to their default modes — only to a perturbed version of them.
- If causal attribution in the brain's recurrent dynamics is not merely hard but principled-impossible at the timescales the user cares about, and the interface can never offer better-than-random causal stories, then the tool surfaces correlated activity without explanation, which is closer to noise than to insight.
- If confabulation is constitutive of conscious self-knowledge rather than a bug to be removed, and the conscious workspace cannot integrate raw process telemetry as "mine" but instead routes around it the way the brain routes around alien thoughts in source-monitoring failure, then the readout sits unintegrated and the user does not become more self-knowing, only more self-data-rich.
- If readout bandwidth, after the most aggressive selection the interface can defend, still exceeds working memory's capacity to use it, the interface produces a permanent backlog the user can never catch up on, and the practical effect is to make cognition feel always-behind-the-readout.
- If the decoder's labels are not stable under ordinary neural drift — if the patterns that meant "fear response" last month no longer mean it this month, and continuous retraining cannot keep up — then the interface produces a self-image whose lexicon changes underfoot, and "I have learned to read myself" becomes a temporary state the user cycles in and out of.

## Open questions

- What is the right functional ontology for the decoder? The labels carry assumptions about what the brain does; the wrong ontology forces real activity into mis-fitting categories. Does the interface impose a chosen ontology, discover one empirically per user, or some negotiated middle?
- Is the readout legible to other people with consent, or necessarily idiosyncratic to one brain? If transferable, introspection becomes interpersonal in a way concept-level IO does not — it transmits process, not content.
- Does sustained access to one's own cognitive process change the process itself on a longer timescale than the moment-to-moment observer effect? Years of legible cognition probably produce a different person than years of confabulated cognition, and the direction of the change is not obvious.
- When the readout disagrees with introspective report — the user says "I am not angry," the readout says amygdala-tagged-orienting-on-spouse — which is authoritative for the user's purposes, and how does the user revise self-knowledge when the disagreement is durable?
- Can the readout be deceived? An adversary with write access to the decoder, or to the labeled state space, has a channel to give the user a false image of themselves. Detecting that the readout is wrong is hard precisely because the user is relying on it for self-correction.
- Is there a meaningful baseline introspection-quality measurement against which the interface's contribution can be assessed, or is the comparison structurally one-sided because pre-interface introspection has no ground truth to be wrong against?

## Risks

- Self-image collapse. The user's narrative self-understanding is the product of decades of confabulation, much of which is sustaining. Replacing it abruptly with telemetry that contradicts the narrative — "the person I thought I was is not present in the readings" — has no graceful transition path, and the failure mode is depersonalization rather than insight.
- Surveillance at the substrate level. A readout exposed to the user can be exposed to anyone who controls the interface. The class of information visible to an employer, court, or state with access to the telemetry stream is qualitatively beyond what behavior or speech reveals, and consent to introspection is not consent to its disclosure.
- Affective avoidance and its inverse. The brain has reasons it does not surface every signal to consciousness; sustained attention to amygdala activity, pain telemetry, or grief processing may impair the functions that depend on those processes running unwatched. The opposite failure is also live: a user who finds the readout unpleasant learns to suppress attention to it, and the interface becomes a thing they have but do not use.
- Telemetry-mediated coercion. An adversary writing to the decoder, or controlling its labeling, gives the user a falsified self-model. Because the user has come to rely on the readout for self-knowledge, falsified telemetry becomes a more effective coercion vector than falsified information about the external world.
- Action-on-readout addiction. The user learns to act on what the readout says rather than on integrated judgment in cases where integrated judgment would have been correct. The readout's abstraction is not the substrate; over-trusting it makes the user worse at integrating tacit signals the readout does not name.
- Loss of healthy opacity. Some self-knowledge gaps are protective. The brain does not surface the felt content of a phobia at full intensity outside the triggering situation. Continuous access removes those gaps, and the user lives with the felt content of every disposition simultaneously, with no off-switch corresponding to the natural quieting that opacity provides.
